================================================================================
                          API ENDPOINTS SUMMARY
================================================================================

Listings:
  POST   /api/v1/listings              - Create listing
  GET    /api/v1/listings              - Search listings
  GET    /api/v1/listings/featured     - Get featured listings
  GET    /api/v1/listings/:id          - Get listing detail
  PATCH  /api/v1/listings/:id          - Update listing
  DELETE /api/v1/listings/:id          - Delete listing

Orders:
  POST   /api/v1/orders                - Create order
  GET    /api/v1/orders/me             - Get buyer orders
  GET    /api/v1/orders/:id            - Get order detail
  PATCH  /api/v1/orders/:id/confirm-delivery - Confirm delivery

Wallet:
  GET    /api/v1/wallet                - Get balance
  GET    /api/v1/wallet/transactions   - Get transaction history

Escrow:
  GET    /api/v1/escrow/:orderId       - Get escrow status
  PATCH  /api/v1/escrow/:orderId/release - Release funds
  PATCH  /api/v1/escrow/:orderId/refund - Refund funds

Reviews:
  POST   /api/v1/reviews               - Create review
  GET    /api/v1/reviews/listing/:listingId - Get listing reviews
  PATCH  /api/v1/reviews/:id/respond   - Seller response

Disputes:
  POST   /api/v1/disputes              - Create dispute
  GET    /api/v1/disputes/open         - Get open disputes
  GET    /api/v1/disputes/:id          - Get dispute detail
  PATCH  /api/v1/disputes/:id/resolve  - Resolve dispute

Messages:
  GET    /api/v1/messages              - Get conversations
  GET    /api/v1/messages/unread-count - Get unread count
  GET    /api/v1/messages/conversation/:conversationId - Get messages
  POST   /api/v1/messages/conversation/:conversationId/send - Send message

Notifications:
  GET    /api/v1/notifications         - Get notifications
  GET    /api/v1/notifications/unread  - Get unread
  PATCH  /api/v1/notifications/:id/read - Mark as read

Support:
  POST   /api/v1/support/tickets       - Create ticket
  GET    /api/v1/support/tickets/me    - Get user tickets
  PATCH  /api/v1/support/tickets/:id/resolve - Resolve ticket

Admin:
  GET    /api/v1/admin/dashboard       - Dashboard stats
  GET    /api/v1/admin/listings/pending - Pending listings
  PATCH  /api/v1/admin/listings/:id/approve - Approve listing
  PATCH  /api/v1/admin/listings/:id/reject - Reject listing
  GET    /api/v1/admin/fraud/flagged-listings - Fraud alerts
  GET    /api/v1/admin/analytics/revenue - Revenue analytics

Auth:
  POST   /api/v1/auth/register         - Register user
  POST   /api/v1/auth/login            - Login user
  GET    /api/v1/auth/me               - Get current user
  POST   /api/v1/auth/verify-email     - Verify email
  POST   /api/v1/auth/reset-password   - Reset password

Sellers:
  POST   /api/v1/sellers               - Create seller account
  GET    /api/v1/sellers/me            - Get seller profile
  GET    /api/v1/sellers/:id           - Get seller profile
  PATCH  /api/v1/sellers/me            - Update profile

Users:
  GET    /api/v1/users/me              - Get profile
  PATCH  /api/v1/users/me              - Update profile
  POST   /api/v1/users/avatar          - Upload avatar
  GET    /api/v1/users/me/stats        - Get user stats

Payments:
  POST   /api/v1/payments/webhook/stripe - Stripe webhook
  POST   /api/v1/payments/webhook/flutterwave - Flutterwave webhook
  POST   /api/v1/payments/webhook/paypal - PayPal webhook

================================================================================
                      HOW TO DEPLOY & RUN
================================================================================

LOCAL DEVELOPMENT:

1. Backend:
   cd backend
   npm install
   npx prisma migrate dev
   npm run start:dev

2. Frontend:
   cd frontend
   npm install
   npm run dev

3. Environment Variables:
   - SUPABASE_URL
   - SUPABASE_SERVICE_ROLE_KEY
   - DATABASE_URL
   - JWT_SECRET
   - NEXT_PUBLIC_API_URL

PRODUCTION DEPLOYMENT:

Frontend (Vercel):
   - Push to main branch
   - Auto-deploys to https://velxo.vercel.app
   - Set NEXT_PUBLIC_API_URL to backend URL

Backend (Render):
   - Push to main branch
   - Auto-deploys to https://velxo.onrender.com
   - Set environment variables in Render dashboard
   - PostgreSQL connected via DATABASE_URL
